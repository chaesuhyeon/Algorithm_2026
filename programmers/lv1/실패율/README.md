## [level 1] 실패율
[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/42889)

### 1. 문제 접근 방식
- 현재 어떤 stage에 몇명이 도전하고 있는지를 담는 배열을 구한 뒤에 실패율을 구한다.
- return 값이 stage라서 map에다가 stage를 key값으로해서 해당 stage의 실패율을 value값으로 담아줬다.
- 여기서... 값은 구했으나 순서대로 key값을 어떻게 정렬해서 보여줄 것인가를 고민했는데 모르겠어서 방법만 검색했다.
- java의 entrySet이라는 자료구조를 몰라서 애먹었다. 2시간 넘게걸림..ㅋㅋㅋ 
- 그리고 total이 0이되면 분모가 0이되어 오류가 발생하는 경우도 체크하지 못했다. 다시 풀어볼것...

### 2. 성능 요약
- 실행 시간: 평균 약 6.9ms (최소 3.25ms / 최대 13.16ms)
- 메모리 사용: 평균 약 88.6MB (최소 72.6MB / 최대 101MB)

### 3. 시간 복잡도 분석
N은 stage개수 M은 stages의 길이
- 각 스테이지 도전자 수를 계산할 때: O(N+M)
- 이후 스테이지별로 실패율을 계산하는데 필요한 시간 복잡도: O(N)
- 실패율을 기준으로 스테이지를 정렬할 때: O(NlogN)

이 연산들을 모두 고려하면 O(2*N + M + NlogN)  
=> 최종 시간 복잡도 O(M+NlogN)

## [level 2] 괄호 회전하기
***
[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/76502)

### 문제 접근 방식
- 기존 괄호 문제랑 별 차이 없어보여서 주어진 s문자열만 잘 회전시키면 될 것 같았다.
- 열린 괄호는 스택에 넣어주고, 닫힌 괄호일 경우 pop해서 맞는 짝인지 검사하고, 맞는 짝이 아니라면 스택에 다시 넣어줬다.
- 닫힌 괄호만 있는 문자열의 경우에는 stack.isEmpty()하면 true라서 answer +1이 될 수있기 때문에 그걸 방지하기 위해 success라는 boolean값을 선언해줬다.

### 성능 요약
- 실행 시간: 평균 약 64.0ms (최소 11.05ms / 최대 90.04ms)
- 메모리 사용: 평균 약 86.0MB (최소 72.8MB / 최대 93.6MB)

### 시간 복잡도 분석
N은 s의 길이
- 회전한 괄호 문자열의 유효성을 체크할 때 이중 반복문을 활용 : O(N^2)

** 참고  
push()와 pop()은 시간복잡도가 O(1)이다.

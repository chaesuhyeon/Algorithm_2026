## [level 2] 행렬의 곱셈
***
[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/12949)

### 문제 접근 방식
- 그냥 행렬 문제 풀듯이 풀었다.
- 인덱스에 애를 먹었다.. 인덱스 오류가 계속 떠서 문제푸는데 2시간 넘게 걸렸다.. 다시 풀어보자...
- 행렬의 크기?를 결정하는건 arr2 행렬의 열 개수였는데 arr2 배열의 길이로 하니까 계속 틀렸었음..

** 참고
두 배열의 최대 데이터 개수가 100개이므로 시간 복잡도는 신경 안써도 된다.

### 성능 요약
- 실행 시간: 평균 약 2.40ms (최소 0.09ms / 최대 6.25ms)
- 메모리 사용: 평균 약 86.7MB (최소 74.0MB / 최대 97.2MB)


### 시간 복잡도 분석
N은 행 또는 열의 길이이며 행과 열의 길이는 같다.
- arr1 행의 수 * arr1 열의 수 * arr2 열의 수 만큼 연산함

=> 최종 시간 복잡도는 O(N^3)

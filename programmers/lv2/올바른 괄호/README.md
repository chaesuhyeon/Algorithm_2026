## [level 2] 올바른 괄호
***
[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/12909)

### 문제 접근 방식  
- 많이 풀어봤던 형식이라 스택 문제인걸 바로 알았다.
- 괄호가 '('까지 무조건 사라져야하기 때문에 스택에 '(' 열린 괄호는 무조건 담고 ')' 닫힌 괄호의 경우만 생각했다.
- ')'닫힌 괄호가 나올 경우 스택이 비었으면 이미 짝이 안맞기 때문에 바로 return false를 해줬고 '('괄호가 나온다면 짝이 맞기 때문에 pop을 해줬다.
- 최종적으로 스택이 비었다면 올바른 괄호 짝이기 때문에 answer를 그대로 return해주고 스택이 empty가 아닐 경우에 answer를 false로 바꿔준뒤 return 해준다.

### 성능 요약
- 실행 시간
    - 정확성 테스트: 평균 약 0.22ms (최소 0.10ms / 최대 0.34ms)
    - 효율성 테스트: 평균 약 17.45ms (17.06ms ~ 17.83ms)

- 메모리 사용
    - 정확성 테스트: 평균 약 83.9MB (최소 76.9MB / 최대 99.2MB)
    - 효율성 테스트: 평균 약 54.9MB (54.6MB ~ 55.1MB)

### 시간 복잡도 분석
N은 s의 길이
- s를 순회하며 괄호의 쌍을 확인 O(N)  

=> 최종 시간 복잡도는 O(N) 

** 참고  
push()와 pop()은 시간복잡도가 O(1)이다.
